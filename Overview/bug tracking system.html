Home Page

  1. Import User

  2. Regiter

  3. Login



Import User

  • Import allows json and XML files to be imported and saved in database.
  • Same Email id is not permitted.
  • Email format should be proper.
  • All fields are mandtory.
  • On successful import, each user has to be provided with a unique id.




Register





  • Register allows imported user to register and generate a password.
  • User should not be already registered.
  • Password should be strongly typed.
  • All fields are mandtory.
  • Email id should match Role.
  • After registering goto login page.




Login



  • User should not be already logged in.
  • Email id and password should match.
  • After successful login goto Bugs Tracking Page.




Bug Tracking Page

  1. Project Manager

  2. Developers

  3. Testers



Project Manager


After clicking on Project

  • Bugs can only be assigned to Developers.
  • Bugs can only be closed if they are marked for closing.

After clicking on Create New Project

  • Start date should be set 2 days later than current date.
  • Project Status to be "In Progress".
  • Developers can only be assigned 1 project
  • Testers can only be assigned max of 2 projects.
  • Testers can be assigned under the same manager.
  • PM can manage max 4 projects.
  • Unique Project id to be generated.


Developers


  • Developers can only view Bugs assigned to them.
  • He/She can mark a bug for closing.


Testers


  • Only Bugs reported by Tester can be viewed by him.

After clicking on Report New Bug

  • Bugs can be reported on,y by Testers.
  • Report can be made only for Projects assigned and for projects with status "in Progress".
  • Unique bug id to be generated.
  • Created by should be Tester.
  • Created on should be current date and time.
